# 分析方法筆記

## 1. 原始資料格式

每一行是一筆 cascade：

```text
cascade_id    root_id    timestamp    count    paths
```

`paths` 中每個 token 類似：

```text
root/user_a/user_b:3655
```

意思是 user_b 在原始發文後 3655 秒轉發，傳播路徑是 root 到 user_a 再到 user_b。

## 2. 觀察窗與預測窗

- 觀察窗：前 1 小時，也就是 `delay < 3600`
- 預測窗：24 小時，也就是 `delay < 86400`
- 簡報中的主要預測目標是 `actual_24h_total`

注意：這裡預測的是 24 小時總傳播規模，不是 1 小時後的新增量。CasFlow 原始程式常用的是 incremental popularity，也就是：

```text
24h 內總量 - 觀察窗內總量
```

本報告選擇 24 小時總規模，是因為主題是「早期訊號是否能預警最終爆紅規模」。

## 3. 長尾現象

長尾分析使用完整 Weibo cascade summary：

- cascade 數：119,313
- 前 1% cascade 貢獻約 43.83% 的傳播路徑
- 前 10% cascade 貢獻約 75.33% 的傳播路徑

這支持簡報中的第一個論點：微博傳播不是平均事件，而是少數事件主導大量擴散。

## 4. Baseline 模型

baseline 是線性回歸：

```text
X = 前 1 小時傳播數
Y = 24 小時總傳播規模
```

結果：

- 測試集 R2：0.7471
- MAE：149.84
- RMSE：1068.95

這個模型只應該被解讀為最低參考線。它證明前 1 小時訊號有預警能力，但不代表線性關係足以穩定預測所有爆紅事件。

## 5. 為什麼散點圖是扇形

散點圖不是完美線性，而是越往右越分散。這代表：

- 大多數 cascade 很小，所以左下角點很密集
- 大型 cascade 的誤差變大
- 同樣早期熱度，後續可能停止，也可能二次擴散

這種現象可稱為異質變異，也就是模型誤差會隨著傳播規模放大。

## 6. 改進方向

簡報中的改進方向分成三類：

1. log scale：
   - 使用 `log1p(target)` 訓練模型
   - 降低極端值對模型的拉扯

2. 非線性模型：
   - 使用 HistGradientBoosting
   - 捕捉時間切片、結構特徵、成長比例之間的非線性關係

3. 爆紅分類 + 回歸：
   - 先估計 cascade 是否可能進入 Top 10%
   - 再對傳播規模做回歸

評估不能只看 R2，也要看：

- MAE
- RMSE
- log-RMSE
- Top 10% recall
- 小 / 中 / 大 cascade 分段誤差
